## 3. 关键路径分析 (Critical Path)

**最小可演示版本 (MVP) 时间线：**

```
第1周: 基础工程 (INFRA-001 ~ INFRA-004, BE-001 ~ BE-006, FE-001 ~ FE-006)
    ↓
第2-3周: 认证 (BE-003, FE-003)、初始化数据管理 (MOD-A1, MOD-A2, MOD-A3)
    ↓
第4-5周: 支撑矩阵、课程大纲、考核点 (MOD-A4, MOD-B1 ~ MOD-B3)
    ↓
第6-7周: 核心计算引擎 (MOD-C1 ~ MOD-C4) ← **最复杂且最关键**
    ↓
第8周+: 报表与优化 (MOD-D1, MOD-D2)、性能调优
```

## 6. PR流程规范

### 6.1 分支命名规范

```
feature/MOD-A1-graduation-requirement   # 新功能
bugfix/ISSUE-123-login-token-expired   # 问题修复
refactor/BE-001-auth-framework         # 重构
```

### 6.2 Pull Request检查清单

- [ ] 功能完整性 ✓
- [ ] 单元测试通过 ✓
- [ ] 代码覆盖率 ≥ 80% ✓
- [ ] 遵循编码规范 ✓
- [ ] 无未解决的TODO注释 ✓
- [ ] 数据库迁移脚本就位 ✓
- [ ] 文档已更新 ✓
- [ ] 至少一名代码审查员批准 ✓

### 6.3 合并策略

- 使用 **Squash and Merge** 保持主分支提交历史清晰
- 不允许直接向 `main` 和 `develop` 推送代码
- 严格遵循PR流程，所有修改必须经过代码审查

## 7. 云服务器部署申请

### 申请内容

需要申请一个云服务器用于系统演示部署：
- **OS**: Ubuntu 22.04 LTS
- **内存**: ≥ 4GB RAM
- **存储**: ≥ 50GB SSD
- **CPU**: ≥ 2核
- **软件栈**: Java 17, MySQL 8.0, Nginx, Docker

### 部署步骤

1. 配置服务器环境
2. 构建Docker镜像
3. 编写docker-compose部署文件
4. 配置Nginx反向代理
5. SSL证书配置
6. 性能监控告警
