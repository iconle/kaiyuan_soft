# 项目说明文档

## 项目名称
面向专业认证的毕业要求达成度统一计算平台

## 项目概述
本系统旨在为本科工程教育认证与师范类专业认证提供高精度、可追溯的数据计算支撑。通过结构化的数据流转，实现从"底层考核点原始成绩录入"到"顶层毕业要求达成度输出"的全自动化计算。

## 核心业务价值
- **摒弃手工计算**: 彻底摒弃繁杂的手工Excel计算
- **数据可追溯**: 支持穿透式查询，追溯任意一个最终指标点到具体学生的具体成绩
- **多层级权重**: 实现"宏观矩阵-微观大纲"相分离的双层权重设计
- **自动化流程**: 从成绩录入到计算结果的完全自动化

## 四层权重计算模型

### 第一级：课程目标级达成度（基于考核点得分）
$$C_{ij} = \frac{\sum (\text{支撑目标 } j \text{ 的考核点实际得分})}{\sum (\text{支撑目标 } j \text{ 的考核点目标满分})}$$

班级课程目标 $j$ 的达成度为全体学生 $C_{ij}$ 的算术平均值。

### 第二级：课程级达成度（结合大纲内部贡献权重 $w$）
$$E_{k} = \sum_{j} \left( \bar{C_j} \times w_{jk} \right)$$

约束：针对同一个指标点，$\sum w_{jk} = 1.0$

### 第三级：专业级达成度（结合宏观总支撑权重 $W$）
$$G_{k} = \sum \left( E_{k} \times W_{c} \right)$$

约束：所有支撑同一指标点的课程，$\sum W_{c} = 1.0$

## 关键设计决策

### 1. 双层权重分离设计
- **宏观层 (Support Matrix)**: 由专业负责人维护，定义课程与指标点的支撑关系及总权重
- **微观层 (Syllabus Weights)**: 由课程教师维护，定义目标对指标点的贡献权重
- **好处**: 降低耦合度、提高系统灵活性、支持权重的独立迭代

### 2. 计算触发的两阶段设计
- **阶段一 (教师级触发)**: 课程目标级 + 课程级计算，结果仅锁定本课程
- **阶段二 (全局级触发)**: 专业级汇总计算，由教务/专业负责人统一触发
- **好处**: 提高并发效率、支持增量计算、防止中间过程被干扰

### 3. 强类型语言 + 成熟企业框架选型
- 采用Java + Spring Boot 是为了支撑复杂的权重矩阵校验与高并发Excel解析
- 相比Python/Node.js，Java提供更强的类型安全性和大规模系统稳定性

### 4. 基于成绩的"直接评价"
- 系统不支持问卷、间接评价等其他评价方式
- 所有数据来源于学生成绩，确保数据的客观性与可审计性
